Convention d'Écriture Php

Convention de nommage des variables et objets du langage Php en utilisant des préfixes pour identifier les principaux types (adaptation personnelle de la fameuse Notation Hongroise).

Les Variables

Pour le nommage des variables, mes habitudes d'écriture s'inspirent de la syntaxe CamelCase, par exemple : "sMaJoliePetiteVariable".

Les termes entre parenthèses désignent les valeurs à utiliser pour les transtypages, par ex. $iTest = (int)$sTest;.

Dans les codes d'exemple, comme $bNom, le terme Nom désigne le nom qu'on donnera à la variable.

Booléens

$bNom = boolean (bool) : variables de type booléenne, par ex. true ou 1 et false ou 0 (zéro).

Nombres

$nNom = number : notation générale des variables de type nombre, pouvant être indifféremment de type integer ou float (aussi nommée real).

Pour préciser le type de nombre on peut les noter de la manière suivante :

$iNom = integer (int) : pour les nombres entiers, positifs ou négatifs, par ex. 4 ou -25 ;
$fNom = float (float) : pour les nombres décimaux, positifs ou négatifs, par ex. 4.8 ou -0.256.

Pour éviter d'éventuelles confusions, on peut aussi noter les préfixes composés, par ex. $niNom ou $nfNom, en commençant toujours par le type générique.

Textes

$sNom = string (string) : notation générale des variables de type texte, par ex. 'mon petit texte'.

Pour désigner certaines valeurs habituelles on peut les préciser de la manière suivante :

$dNom = date : pour les dates et les heures ;
$hNom = html : pour les codes Html ;
$uNom = url : pour les liens Url de type "http", "ftp", "mailto" etc ;
$pNom = path : pour les chemins d'accès aux fichiers ou dossiers.

Pour éviter d'éventuelles confusions, on peut aussi noter les préfixes composés, par ex. $sdNom, $suNom, etc., en commençant toujours par le type générique.

Tableaux

$aNom = array (array) : variables de type tableau, simples ou associatifs, par ex. array('nom'=>'Fredo', 'age'=>40).

Dans les tableaux associatifs, les clés peuvent aussi suivre les mêmes recommandations de nommage que celles préconisées pour les variables, par ex. array('sNom'=>'Fredo', 'iAge'=>40).

Ressources

$rNom = resource : pour les ressources externes, comme les fichiers de la fonction fopen().

Objets

$oNom = object (object) : instances d'une classe obtenues par la commande new.

Les Fonctions et Classes

Pour le nommage des fonctions et classes, pour bien les distinguer des variables, mes habitudes d'écriture s'inspirent de la syntaxe Underscore, par exemple : "f_ma_jolie_petite_fonction()".

f_nom = function : uniquement pour les fonctions utilisateur (vs fonctions du langage) ;
c_nom = class : classes d'objets définies par l'utilisateur.

Les Cas Particuliers

On peut également avoir besoin d'indiquer la portée ou le comportement de certaines variables, comme les globales et superglobales, ainsi que les statiques, voici donc quelques suggestions...

Statiques

Les variables statiques (static) peuvent être distinguées des autres en ajoutant, après les autres préfixes et juste avant le nom de la variable, le préfixe "t".

$tNom = static : à ajouter juste devant le nom de la variable, par ex. $itNom, $stNom, etc.

Globales

Les variables globales peuvent être distinguées des autres en ajoutant au début le préfixe "g".

$gNom = global : à placer devant le type de variable, par ex. $guNom, $gsNom, etc.

Superglobales

Pour les valeurs issues des variables superglobales, il est aussi possible de les distinguer, par des préfixes correspondants suivis d'un tiret bas (underscore) '_', par ex. $get_sNom (dans ce cas, le préfixe 'g' n'est pas nécessaire).

$env_ = $_ENV : variables d'environnement ;
$srv_ = $_SERVER : variables issues du serveur ;
$get_ = $_GET : variables passées par la méthode 'get' ;
$pst_ = $_POST : variables passées par la méthode 'post' ;
$req_ = $_REQUEST : variables issues des requêtes ;
$cok_ = $_COOKIE : variables issues des 'cookies' ;
$ses_ = $_SESSION : variables issues des 'sessions' ;
$glb_ = $GLOBALS : variables issues des 'globals'.

Les Cas Indéfinis

$nom = aucun préfixe : tous les autres cas de figure non prévus.

Les Personnalisations

Il peut être parfois utile, lorsqu'on travail sur plusieurs projets, d'identifier les variables, fonctions et classes de chaque projet, pour éviter notamment les appéllations identiques sources de disfonctionnements.

Le plus simple pour faire cette distinction est d'attribuer un préfixe spécifique à chaque projet, puis faire précéder chaque élément important de leur code par ce préfixe.

Par exemple, pour un projet qui porterait le nom de "Formulaire Contact", ou pourrait adopter le préfixe "fc", puis il suffirait de l'appliquer aux variables (notamment à celles qui sont globales) : $fc_gNom, aux fonctions et classes : fc_f_Nom et fc_c_Nom.

Enfin, tous les fichier qui composent un projet peuvent aussi en bénéficier : fc_config.php.
